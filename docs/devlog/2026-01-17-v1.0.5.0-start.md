## Development Plan

### Phase 1: Toast Notification System (2-3 days)

**Location:** `/app/src/components/`

- ToastContainer.svelte - Main container component
- Toast.svelte - Individual notification component
- Notification types: info, success, warning, error, progress
- Auto-dismiss timing (configurable, default 5s)
- Action buttons support (primary, secondary)
- Animation: Slide in/out

**Files to Create:**

- `/app/src/components/Toast/ToastContainer.svelte`
- `/app/src/components/Toast/Toast.svelte`
- `/app/src/components/Toast/types.ts`
- `/app/src/components/Toast/Toast.module.css` (or Tailwind)

### Phase 2: Notification Store (2 days)

**Location:** `/app/src/stores/`

- Writable store for notification queue
- Methods: add(), remove(), clear()
- Queue management (max 10 notifications)
- Auto-dismiss handling
- Persistent history (SQLite optional)

**Files to Create:**

- `/app/src/stores/notifications.ts`
- `/app/src/lib/notification-service.ts`

### Phase 3: Slack Integration (3-4 days)

**Location:** `/wizard/services/`

**API Endpoints:**

- POST `/api/slack/send` - Send notification
- GET `/api/slack/channels` - List channels
- POST `/api/slack/thread` - Reply in thread
- GET `/api/slack/config` - Check configuration

**Files to Create:**

- `/wizard/services/slack_service.py` (250-300 lines)
- `/wizard/routes/slack_routes.py` (150-200 lines)

### Phase 4: Notification History (2 days)

**Location:** `/memory/notifications.db` + `/app/src/`

- SQLite schema (notifications table)
- Export to JSON, CSV, Markdown
- UI component for history viewer
- Filter/search support

### Phase 5: Documentation & Testing (2-3 days)

- Create `docs/v1.0.5.0-INTEGRATION-GUIDE.md`
- Create `docs/howto/SLACK-INTEGRATION.md`
- Create `docs/howto/TOAST-NOTIFICATIONS.md`
- Test procedures for all components
