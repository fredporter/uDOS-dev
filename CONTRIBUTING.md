# Contributing to uDOS-dev Scaffold

Thanks for your interest in improving the uDOS development scaffold!

---

## Two Types of Contributions

### 1. \ud83d\udcdd Scaffold Framework (This Repo)

**For template improvements, documentation, and scaffold structure:**

- **Repository:** [uDOS-dev](https://github.com/fredporter/uDOS-dev)
- **Issues:** [Report scaffold issues](https://github.com/fredporter/uDOS-dev/issues)
- **What to contribute:**
  - Template scripts and build configurations
  - Example code and reference implementations
  - Wiki documentation improvements
  - Scaffold structure enhancements

**Guidelines:**
- Keep it minimal \u2014 framework only, no specific projects
- Make templates generic and adaptable
- Document usage patterns clearly
- No private or project-specific code

### 2. \ud83d\udee0\ufe0f Main uDOS Project

**For core features, bugs, and main project improvements:**

- **Repository:** [uDOS](https://github.com/fredporter/uDOS)
- **Contributing Guide:** [CONTRIBUTORS.md](https://github.com/fredporter/uDOS/blob/main/CONTRIBUTORS.md)
- **Issues:** [Report main project issues](https://github.com/fredporter/uDOS/issues)
- **Wiki:** [CONTRIBUTING](https://github.com/fredporter/uDOS/wiki/CONTRIBUTING)

## Quick Start

```bash
# Fork the repo
git clone https://github.com/YOUR-USERNAME/uDOS-dev.git
cd uDOS-dev

# Create a branch
git checkout -b feature/your-improvement

# Make your changes
# - Add templates to /scripts/ or /tools/
# - Update wiki in /wiki/
# - Add examples to /examples/

# Commit and push
git add .
git commit -m "Add: description of your improvement"
git push origin feature/your-improvement

# Open a PR on GitHub
```

---

## PR Checklist

- \u2610 Changes are generic and reusable
- \u2610 No project-specific or private code included
- \u2610 Documentation updated if needed
- \u2610 Examples are clear and well-commented
- \u2610 Templates follow uDOS [Style Guide](https://github.com/fredporter/uDOS/wiki/STYLE-GUIDE)
